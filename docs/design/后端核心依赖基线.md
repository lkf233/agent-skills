# 后端核心依赖基线（V1）

## 1. 目标与范围

- 适用范围：`backend/` 单体后端（Spring Boot + Maven + PostgreSQL）。
- 目标：为登录注册、会话管理、历史记忆、知识库、MCP 混合模式提供统一依赖基线。
- 强约束：同类能力只允许一种主方案，避免多框架并行引发维护和冲突成本。
- 当前阶段策略：V1 采用轻量鉴权与基础可用优先，不引入复杂安全监测体系。

## 3. 同类能力唯一方案

- ORM 方案唯一：MyBatis；禁用 JPA/Hibernate 并行。
- HTTP 客户端唯一：优先 Spring `RestClient`，禁用 `OkHttp + Apache HttpClient + Hutool Http` 混用。
- JSON 库唯一：Jackson；禁用 Gson/Fastjson 并行。
- 校验框架唯一：Jakarta Validation；禁用手写重复注解体系。
- 日志框架唯一：Log4j2；禁用多日志实现并存。
- 鉴权方案唯一（V1）：JWT + HandlerInterceptor；暂不引入 Spring Security 复杂过滤器链。

## 4. 替代策略与技术债标记

- 替代策略：若需替换核心依赖，必须先提供“必要性、替代对比、迁移范围、回滚路径”四项说明。
- 技术债标记：在需求单与 PR 标记 `DEPENDENCY_DEBT`，并附截止版本与清理时间。
- 临时并存策略：只允许在迁移窗口内短期并存，不超过两个迭代。

## 5. 版本治理与升级策略

### 5.1 版本治理规则

- 统一使用父 POM + `dependencyManagement` 锁定核心版本。
- 子模块禁止覆盖核心依赖版本，仅允许新增非核心依赖。
- 每月第 2 周作为升级窗口，窗口外仅允许安全热修。

### 5.2 升级验证清单

- 启动验证：应用可启动，健康检查 `UP`。
- 核心接口：认证、会话、知识库、MCP 配置接口可用。
- 数据连接：迁移脚本可执行，连接池与事务正常。
- SSE 验证：对话流可持续输出 token/step/tool 事件。
- MCP 调用：内置工具与用户配置工具各至少 1 条用例通过。
- 回滚策略：保留前一版本镜像与数据库迁移回滚脚本。

## 6. 最小可运行依赖集

### 6.1 首批必选（MVP）

- Spring Boot：web、validation、actuator。
- 数据层：mybatis、postgresql、flyway、pgvector。
- AI 层：langchain4j、langchain4j-open-ai、langchain4j-pgvector。
- 工程层：log4j2、starter-test。

### 6.2 满足的首批能力

- 认证：注册登录、JWT 鉴权（轻量拦截器方案）。
- 会话：会话创建、分页查询、消息持久化。
- 历史记忆：短期摘要与长期偏好写入。
- 知识库：文档切分、向量化入库、检索召回。
- MCP：内置工具调用 + 用户配置工具调用。

## 7. 扩展依赖集与引入门禁

| 扩展能力 | 候选依赖 | 引入门禁（必须同时满足） |
| --- | --- | --- |
| 缓存 | `spring-boot-starter-data-redis` | 明确性能瓶颈、给出本地缓存替代评估、说明失效策略与降级 |
| 消息队列 | `spring-kafka` / `spring-boot-starter-amqp` | 说明异步必要性、幂等设计、失败重试与死信策略 |
| 异步任务编排 | `shedlock` / `jobrunr` | 给出任务可观测方案、并发控制、手工补偿方案 |
| 追踪监控 | `micrometer-tracing-bridge-otel` | 明确采样率、数据保留策略、隐私脱敏方案 |

- 每次扩展引入都必须附带回滚方案：依赖移除步骤、配置回退步骤、数据兼容说明。

## 8. 安全与合规基线（简化版）

- V1 暂不建设安全监测平台与自动漏洞流水线。
- V1 仅保留基础安全要求：密码哈希存储、输入参数校验、敏感信息日志脱敏。
- 许可证检查与漏洞扫描在 V2 纳入 CI 规范。

## 11. 执行约束

- 本文档为后端依赖准入基线，新增同类依赖默认不予通过。
- 如确需破例，必须完成“门禁评审 + 风险登记 + 回滚预案”。
- V1 不引入复杂安全监测与复杂 Security 鉴权框架，后续按业务复杂度再升级。
